OBJETIVO:

O objetivo deste projeto foi implementar um processo de qualidade de testes funcionais que garanta a qualidade dos testes, assegurando que as aplicações funcionem conforme o esperado e detectando bugs durante o processo de desenvolvimento. Para alcançar esse objetivo, foi proposta a metodologia BDD, na qual os requisitos de negócios são definidos como Critério de Aceitação e os requisitos funcionais como Cenários de Testes, vinculados às histórias de usuários.
Os projetos eHealth Screen Module e Workflow Stroke HC foram selecionados para participar deste projeto. Para cada um desses projetos, foram realizados testes de navegabilidade (Front End) e testes de análise de dados e cálculos. A metodologia BDD foi aplicada de forma a garantir a qualidade dos testes e permitir a detecção precoce de erros, contribuindo para a entrega de aplicações mais robustas e confiáveis.

PROBLEMA TÉCNICO-CIENTÍFICO:

O problema a ser solucionado era garantir o entendimento dos requisitos de negócios esperados, para que novos releases pudessem ser considerados concluídos e implementados. Para solucionar esse problema, foi necessário definir quais requisitos funcionais deveriam ser testados, de forma a atender aos requisitos de negócios mapeados.

Para executar o projeto, foram utilizados diversos recursos, incluindo análise das documentações funcionais, mapeamento de cenários, modelagem dos cenários de testes, execução de testes manuais, apontamento de erros e melhorias, acompanhamento de correções e retestes, envio de relatórios (evidências) e status report semanal. Essas etapas permitiram identificar e solucionar os problemas relacionados aos requisitos de negócios e garantir que os releases fossem implementados com sucesso, atendendo às expectativas do projeto e garantindo a qualidade das aplicações entregues.
